### Глоссарий современных технологий и продуктов для DevOps и прочих IT специалистов.

### A

- **AB Testing** - Метод сравнения производительности новой версии модели (A) со старой (B) на части трафика для принятия решения о полном развертывании.

- **Access Control List (ACL)** - список разрешений, определяющий права доступа к объектам.

- **ACID (Atomicity, Consistency, Isolation, Durability)** - Набор свойств транзакций в базах данных, гарантирующий надежность: Атомарность, Согласованность, Изолированность, Долговечность.

- **Active Directory (AD) (Microsoft)** - служба каталогов для управления пользователями и компьютерами в сети Windows.

- **Agile** - гибкая методология разработки программного обеспечения.

- **AI (Artificial Intelligence)** - моделирование человеческого интеллекта машинами.

- **AirFlow (Apache)** - платформа для оркестрации рабочих процессов на Python.

- **Airbyte (Airbyte)** - Платформа с открытым исходным кодом для интеграции данных (ELT), позволяющая легко переносить данные из сотен источников в хранилища, озера и базы данных.

- **AJAX (Asynchronous JavaScript and XML)** - подход к созданию интерактивных веб-приложений.

- **Algorithm** - пошаговая инструкция для решения задачи.

- **Amazon SageMaker (Amazon Web Services)** - управляемая облачная платформа для создания и развертывания моделей ML.

- **Anaconda (Anaconda, Inc.)** - дистрибутив Python для data science.

- **Android (Google)** - мобильная операционная система на базе Linux.

- **Angular (Google)** - фреймворк для создания клиентских веб-приложений.

- **Ansible (Red Hat)** - инструмент для автоматизации конфигурации и развертывания.

- **API (Application Programming Interface)** - интерфейс для взаимодействия между программами.

- **Apache** - фонд, поддерживающий множество проектов с открытым исходным кодом.

- **Apache Flink (Apache)** - фреймворк для обработки потоковых и пакетных данных.

- **Apache HTTP Server (Apache)** - популярный веб-сервер с открытым исходным кодом.

- **Apache Spark (PySpark) (Apache)** - распределенный движок для обработки больших данных.

- **AppImage** - универсальный формат пакетов для Linux.

- **Application Server** - серверная платформа для выполнения бизнес-логики приложений.

- **Argo CD (Argo Project)** - инструмент GitOps для непрерывной доставки в Kubernetes.

- **Argo Workflows (Argo Project)** - движок для оркестрации рабочих процессов в Kubernetes.

- **Artificial Neural Network (ANN)** - вычислительная модель, вдохновленная биологическими нейронными сетями.

- **ASCII (American Standard Code for Information Interchange)** - стандарт кодирования символов.

- **[ASP.NET](https://asp.net/) (Microsoft)** - веб-фреймворк для создания динамических сайтов и приложений.

- **Assembly Language** - низкоуровневый язык программирования.

- **Asterisk** - платформа с открытым исходным кодом для построения телефонных коммуникаций.

- **Asynchronous** - выполнение операций без блокировки основного потока.

- **ATX (Advanced Technology eXtended)** - стандарт конструкции материнских плат и блоков питания.

- **Audit Log** - журнал регистрации событий для анализа действий.

- **Authentication** - процесс проверки подлинности пользователя или системы.

- **Authorization** - процесс проверки прав доступа к ресурсам.

- **AutoGPT** - эксперимент по применению GPT для автономного выполнения задач.

- **Automation** - использование технологий для выполнения задач без участия человека.

- **AWS (Amazon Web Services)** - публичная облачная платформа от Amazon.

- **Azure (Microsoft)** - публичная облачная платформа от Microsoft.

### B

- **Backend** - серверная часть приложения, отвечающая за логику и данные.

- **Backport** - перенос исправления из новой версии ПО в старую.

- **Backpressure** - механизм управления потоком данных в системах реального времени.

- **Backstop** - механизм отката изменений при неудачном обновлении.

- **Backstage** (Linux Foundation)** - фреймворк для создания порталов разработчиков (IDP).

- **Backstop Solution** - резервное решение на случай отказа основной системы.

- **Backup** - процесс создания копии данных для восстановления.

- **Backup Window** - период времени, допустимый для создания резервных копий.

- **Backward Compatibility** - способность новой системы работать со старыми данными или интерфейсами.

- **Bamboo (Atlassian)** - сервер CI/CD от Atlassian.

- **Bare Metal** - физический сервер без гипервизора или предустановленной ОС.

- **Bash (Bourne-Again SHell)** - командная оболочка Unix, стандарт в Linux и macOS.

- **Batch Processing** - обработка большого объема данных без интерактивного взаимодействия.

- **BCP (Business Continuity Planning)** - планирование непрерывности бизнеса.

- **Behavior-Driven Development (BDD)** - методология разработки, основанная на поведении системы.

- **BentoML** - Фреймворк для создания, развертывания и масштабирования сервисов машинного обучения в production, ориентированный на производительность и простоту.

- **Big Data** - технологии для работы с огромными и сложными наборами данных.

- **BigQuery (Google)** - бессерверное, масштабируемое облачное хранилище данных.

- **Binary** - файл, содержащий исполняемый машинный код.

- **Binary Large Object (BLOB)** - тип данных для хранения больших двоичных объектов (изображения, видео).

- **BIOS (Basic Input/Output System)** - прошивка для инициализации аппаратного обеспечения при загрузке ПК.

- **Bit** - наименьшая единица информации в вычислительной технике (0 или 1).

- **Bitbucket (Atlassian)** - веб-сервис для хостинга репозиториев Git и Mercurial.

- **BitLocker (Microsoft)** - технология шифрования дисков в Windows.

- **Blockchain** - цепочка блоков для записи транзакций.

- **Blue-Green Deployment** - стратегия развертывания с двумя идентичными средами.

- **Bluetooth** - стандарт беспроводной связи на коротких расстояниях.

- **Bootloader** - программа, загружающая операционную систему.

- **Bootstrap** - популярный CSS-фреймворк для создания адаптивных сайтов.

- **Bot** - автономная программа, имитирующая действия человека.

- **Bottleneck (Узкое место)** - компонент системы, ограничивающий общую производительность.

- **Branch** - ветвь в системе контроля версий для изолированной разработки.

- **Browser** - программа для просмотра веб-страниц (Chrome, Firefox).

- **BSD (Berkeley Software Distribution)** - ветвь операционных систем Unix с открытым исходным кодом.

- **Buffer Overflow** - ошибка, когда программа пишет данные за пределы выделенного буфера.

- **Bug** - ошибка или дефект в программном обеспечении.

- **Build** - процесс преобразования исходного кода в исполняемое приложение.

- **Business Intelligence (BI)** - технологии для анализа бизнес-данных и поддержки решений.

- **Byte** - единица информации, состоящая из 8 бит.

- **Bytecode** - промежуточный код, исполняемый виртуальной машиной (например, в Java).

### C

- **C** - компилируемый язык программирования общего назначения, высокопроизводительный.

- **C# (C Sharp) (Microsoft)** - объектно-ориентированный язык программирования для платформы .NET.

- **C++** - компилируемый язык программирования общего назначения, расширение C с классами.

- **Cache** - высокоскоростная память для временного хранения часто используемых данных.

- **Canary Release** - Стратегия постепенного развертывания новой модели для небольшой, контролируемой группы пользователей перед полным релизом..

- **CAP Theorem (Brewer's Theorem)** - теорема о невозможности одновременно обеспечить согласованность, доступность и устойчивость к разделению в распределенной системе.

- **CD (Continuous Delivery/Deployment)** - практика автоматического развертывания кода в production-среду.

- **CDC (Change Data Capture)** - Технология отслеживания изменений в базе данных-источнике (вставки, обновления, удаления) и их доставки в другие системы для дальнейшей обработки.

- **CDN (Content Delivery Network)** - распределенная сеть серверов для быстрой доставки контента.

- **Ceph** - Распределенная платформа хранения данных с открытым исходным кодом, предоставляющая интерфейсы для объектного, блочного и файлового хранения.

- **CentOS (Red Hat)** - дистрибутив Linux на основе исходного кода Red Hat Enterprise Linux (RHEL).

- **Certificate Authority (CA)** - организация, выпускающая цифровые сертификаты.

- **CI (Continuous Integration)** - практика автоматического тестирования и сборки кода.

- **CI/CD** - практика автоматизации этапов сборки, тестирования и развертывания приложения.

- **Cipher** - алгоритм шифрования для преобразования открытого текста в шифрованный.

- **ClearML (Allegro AI)** - Сквозная платформа MLOps с открытым исходным кодом для управления экспериментами, автоматизации pipeline, развертывания и мониторинга.

- **ClickHouse (ClickHouse, Inc.)** - высокопроизводительная колоночная СУБД.

- **Client** - устройство или программа, запрашивающая услуги у сервера.

- **Cloud Computing** - модель предоставления вычислительных ресурсов через интернет.

- **Cloudflare** - компания, предоставляющая услуги CDN, DNS и защиту от DDoS.

- **Cluster** - группа связанных серверов, работающих как единая система.

- **CMDB (Configuration Management Database)** - база данных для управления информацией об ИТ-активах.

- **CMS (Content Management System)** - система управления содержимым сайта (WordPress, Drupal).

- **Code Review** - проверка исходного кода другими разработчиками.

- **Compatibility** - способность систем или компонентов работать совместно.

- **Compiler** - программа, переводящая код с языка высокого уровня в машинный код.

- **Computer Vision** - область ИИ, позволяющая компьютерам "видеть" и понимать изображения.

- **Concurrency** - выполнение нескольких задач в перекрывающиеся периоды времени.

- **Configuration Drift** - неконтролируемое расхождение конфигураций серверов со временем.

- **Configuration Management** - процесс поддержания систем в желаемом, согласованном состоянии.

- **Consul (HashiCorp)** - инструмент для обнаружения сервисов и конфигурации сетей.

- **Container** - изолированная среда для запуска приложения со всеми зависимостями.

- **containerd (Cloud Native Computing Foundation)** - высокоуровневая среда выполнения контейнеров.

- **Continuous Testing** - практика автоматического тестирования на протяжении всего цикла разработки.

- **Cookie** - небольшой фрагмент данных, отправляемый веб-сервером и хранимый браузером.

- **CPU (Central Processing Unit)** - центральный процессор, "мозг" компьютера.

- **Cron** - планировщик задач в Unix-подобных системах.

- **Cross-Platform** - способность ПО работать на нескольких аппаратных платформах или ОС.

- **Crossplane (Cloud Native Computing Foundation)** - фреймворк для управления облачной инфраструктурой через Kubernetes.

- **CRUD (Create, Read, Update, Delete)** - четыре базовые операции с данными.

- **Cryptocurrency** - цифровая валюта, использующая криптографию (Биткойн, Эфириум).

- **Cryptography** - наука о методах обеспечения конфиденциальности и целостности данных.

- **CSI (Container Storage Interface)** - Стандартный интерфейс для интеграции внешних систем хранения данных с Kubernetes, позволяющий подключать различные хранилища (например, от AWS, Google Cloud, Ceph).

- **CSRF (Cross-Site Request Forgery)** - атака, при которой выполняются несанкционированные команды от имени пользователя.

- **CSS (Cascading Style Sheets)** - язык стилей для описания внешнего вида веб-страниц.

- **Cybersecurity** - практика защиты систем, сетей и программ от цифровых атак.

- **Cython** - язык программирования, упрощающий написание C-расширений для Python.

### **D**

- **DAG (Directed Acyclic Graph)** - Ориентированный ациклический граф. В Data Engineering и Airflow: способ представления рабочего процесса, где задачи являются узлами, а зависимости между ними - ребрами.

- **Dagster** - Платформа для оркестрации рабочих процессов данных (Data Orchestration), где главной единицей является не задача, а граф активов (таблицы, модели, отчеты).

- **Dart (Google)** - язык программирования, используемый для разработки под Flutter.

- **Data Breach** - инцидент, при котором конфиденциальные данные просматриваются или извлекаются без авторизации.

- **Data Engineering** - практика проектирования и построения систем для сбора, хранения и обработки данных.

- **Data Lake** - хранилище, содержащее огромное количество необработанных данных в первоначальном формате.

- **Data Mining** - процесс обнаружения паттернов и знаний из больших массивов данных.

- **Data Science** - междисциплинарная область, использующая научные методы для извлечения знаний из данных.

- **Data Warehouse** - централизованное хранилище структурированных данных для отчетности и анализа.

- **Database** - организованная коллекция структурированных данных, хранящихся электронным способом.

- **Databricks (Databricks)** - унифицированная платформа для данных и ИИ на основе Spark.

- **Dbt (data build tool)** - Трансформационный инструмент, позволяющий аналитикам и инженерам данных превращать данные в хранилище с помощью SQL, управляя преобразованиями, тестированием и документацией как кодом.

- **DDL (Data Definition Language)** - подмножество SQL для определения и изменения структуры базы данных.

- **DDoS (Distributed Denial of Service)** - атака, направленная на перегрузку ресурсов жертвы трафиком с множества источников.

- **Debian** - один из старейших и наиболее влиятельных дистрибутивов Linux.

- **Debugging** - процесс поиска и исправления ошибок (багов) в программе.

- **Decision Tree** - алгоритм машинного обучения для классификации и регрессии в виде древовидной структуры.

- **Deep Learning** - подраздел машинного обучения, использующий многослойные нейронные сети.

- **Dependency** - внешняя библиотека или модуль, от которых зависит работа программы.

- **Dependency Injection (DI)** - шаблон проектирования, при котором объекты получают свои зависимости извне.

- **DevOps** - культурная и профессиональная практика, объединяющая разработку и эксплуатацию.

- **DHCP (Dynamic Host Configuration Protocol)** - сетевой протокол для автоматического назначения IP-адресов устройствам.

- **Digital Signature** - математическая схема для проверки подлинности цифровых сообщений или документов.

- **Disk Partitioning** - разделение жесткого диска на логические секции (разделы).

- **Distributed System** - система, компоненты которой расположены на разных компьютерах, координирующих свои действия.

- **Django (Django Software Foundation)** - высокоуровневый веб-фреймворк на языке Python.

- **DML (Data Manipulation Language)** - подмножество SQL для манипулирования данными (вставка, обновление, удаление).

- **DNS (Domain Name System)** - система, преобразующая доменные имена (например, google.com) в IP-адреса.

- **Docker (Docker, Inc.)** - платформа для разработки, доставки и запуска контейнеров.

- **Docker Compose (Docker, Inc.)** - инструмент для определения и запуска многоконтейнерных приложений.

- **Docker Swarm (Docker, Inc.)** - встроенный в Docker инструмент для оркестрации контейнеров.

- **Documentation** - справочная информация, описывающая использование и архитектуру программного обеспечения.

- **DOM (Document Object Model)** - программный интерфейс для HTML и XML документов, представляющий их в виде дерева объектов.

- **Domain** - группа компьютеров и устройств в сети, управляемых по общим правилам.

- **DRAM (Dynamic Random-Access Memory)** - тип энергозависимой памяти, используемой для оперативного хранения данных (ОЗУ).

- **DRY (Don't Repeat Yourself)** - принцип разработки ПО, направленный на снижение повторения кода.

- **Dual Boot** - конфигурация, позволяющая выбрать одну из двух или более ОС при загрузке компьютера.

- **DVC (Data Version Control) (Iterative)** - Система контроля версий для данных, моделей и экспериментов, построенная поверх Git, но предназначенная для работы с большими файлами.

- **Dynamic Link Library (DLL)** - концепция Microsoft для библиотек, загружаемых во время выполнения программы.

- **Dynamic Programming** - метод решения сложных задач путем разбиения их на более простые подзадачи.

- **E2E Testing (End-to-End Testing)** - тестирование полного рабочего процесса приложения от начала до конца.

- **Eclipse** - интегрированная среда разработки (IDE) с открытым исходным кодом, в основном для Java.

- **Edge Computing** - парадигма обработки данных ближе к источнику их возникновения, а не в централизованном облаке.

- **EDI (Electronic Data Interchange)** - стандартизированный электронный обмен бизнес-документами между организациями.

- **Egress Traffic** - сетевой трафик, исходящий из сети или системы.

- **Elasticsearch (Elastic)** - поисковый и аналитический движок на базе Apache Lucene.

- **Elementary data** - Инструмент с открытым исходным кодом для мониторинга качества данных, профилирования и построения lineage в проектах dbt и хранилищах данных.

- **ELK Stack (Elasticsearch, Logstash, Kibana)** - стек технологий для сбора, обработки, хранения и визуализации логов.

- **Emacs** - расширяемый текстовый редактор с большим количеством функций.

- **Embedded System** - специализированная вычислительная система, встроенная в устройство (например, микроволновку, автомобиль).

- **Encapsulation** - принцип ООП, заключающийся в объединении данных и методов в единый объект и сокрытии деталей реализации.

- **Encryption** - процесс преобразования информации для сокрытия её содержания (шифрование).

- **Endpoint** - устройство или узел на границе сети (компьютер, телефон, сервер).

- **Enum (Enumeration)** - тип данных, представляющий собой набор именованных констант.

- **Envoy (Cloud Native Computing Foundation)** - высокопроизводительный прокси-сервер для сервисных сетей.

- **EPROM (Erasable Programmable Read-Only Memory)** - вид постоянной памяти, стираемой ультрафиолетовым излучением.

- **LAN (Local Area Network)** - локальная вычислительная сеть, ограниченная небольшой географической зоной (офис, здание).

- **Laravel** - бесплатный веб-фреймворк с открытым исходным кодом для создания приложений на PHP.

- **Latency** - задержка, время между отправкой запроса и получением ответа.

- **LDAP (Lightweight Directory Access Protocol)** - протокол прикладного уровня для доступа к службам каталогов.

- **LeetCode** - онлайн-платформа для подготовки к техническим собеседованиям в IT-компаниях.

- **Legacy Code (Легаси-код)** - устаревший код, который сложно поддерживать и модифицировать, но который всё ещё используется.

- **Library (Библиотека)** - набор предварительно написанного кода (функций, классов), который можно использовать в программах.

- **LIFO (Last In, First Out)** - принцип «последним пришёл - первым ушёл», используемый в структуре данных «стек».

- **Linker (Компоновщик)** - программа, которая собирает несколько объектных файлов в единый исполняемый файл.

- **Linux** - семейство Unix-подобных операционных систем с открытым исходным кодом, построенных на одноименном ядре.

- **Linux Kernel (Ядро Linux)** - само ядро операционной системы, созданное Линусом Торвальдсом и сообществом.

- **Load Balancer (Балансировщик нагрузки)** - устройство или программа, распределяющая сетевой трафик между несколькими серверами.

- **Localhost** - стандартное имя хоста, которое указывает на устройство, используемое для доступа к нему (IP-адрес 127.0.0.1).

- **Log (Лог)** - хронологическая запись событий, происходящих в операционной системе или приложении.

- **Logging** - практика сбора и сохранения логов для последующего анализа и отладки.

- **Loki (Grafana Labs)** - Система для агрегации и хранения логов, разработанная для эффективной работы с метками, как в Prometheus.

- **Longhorn (Rancher)** - Распределенное, отказоустойчивое блочное хранилище для Kubernetes, простое в развертывании и управлении.

- **Lua** - легковесный, высокоуровневый, мультипарадигмальный язык сценариев, часто используемый для встраивания в приложения.

- **LXC (Linux Containers)** - технология виртуализации на уровне операционной системы для создания изолированных Linux-сред.

### M

- **Machine Learning (ML)** - область искусственного интеллекта, где системы обучаются на данных для выявления паттернов и прогнозирования без явного программирования.

- **MaaS (Monitoring as a Service)** - модель предоставления услуг по мониторингу через облако.

- **MacOS (Apple)** - проприетарная операционная система для компьютеров Mac.

- **Mainframe (Мэйнфрейм)** - высокопроизводительный компьютер, предназначенный для обработки больших объемов данных и критически важных приложений.

- **Manifest (Манифест)** - файл с метаданными, описывающий конфигурацию или зависимости (например, в Kubernetes, Docker).

- **MariaDB (MariaDB Foundation)** - форк реляционной СУБД MySQL с открытым исходным кодом.

- **Marketplace (Маркетплейс)** - платформа для покупки, продажи и распространения программного обеспечения или облачных сервисов (например, AWS Marketplace, Atlassian Marketplace).

- **Markdown** - облегченный язык разметки для форматирования простого текста.

- **Materialized View (Материализованное представление)** - Объект базы данных, содержащий результаты запроса, физически сохраненные на диске и периодически обновляемые для ускорения сложных отчетов.

- **Matplotlib** - популярная библиотека на Python для создания статических, анимированных и интерактивных визуализаций данных.

- **Maven (Apache)** - инструмент для автоматизации сборки проектов на Java и управления зависимостями.

- **MBR (Master Boot Record)** - специальный сектор на жестком диске, содержащий загрузчик и информацию о разделах.

- **Memcached** - высокопроизводительная распределенная система кэширования объектов в памяти.

- **Memory Leak (Утечка памяти)** - программная ошибка, при которой программа не освобождает память, занятую временными данными.

- **Merge (Слияние)** - операция объединения двух или более веток кода в системах контроля версий.

- **Metadata (Метаданные)** - данные, описывающие другие данные (например, автор файла, дата создания).

- **Metasploit** - фреймворк с открытым исходным кодом для разработки, тестирования и эксплуатации уязвимостей безопасности.

- **MicroK8s (Canonical)** - легковесная, автономная версия Kubernetes.

- **Microservices (Микросервисы)** - архитектурный стиль, при котором приложение состоит из небольших, слабо связанных и независимо развертываемых сервисов.

- **Microsoft Teams (Microsoft)** - платформа для совместной работы, включающая чаты, видеоконференции и интеграцию с Office 365.

- **Middleware (Промежуточное ПО)** - программный слой, обеспечивающий взаимодействие между различными приложениями, компонентами или системами.

- **MinIO** - Высокопроизводительное объектное хранилище, совместимое с API Amazon S3 (Amazon Simple Storage Service), предназначенное для работы с большими данными и искусственным интеллектом.

- **MLflow (Databricks)** - Открытая платформа для управления полным жизненным циклом машинного обучения: эксперименты, реестр моделей, развертывание.

- **MLOps (Machine Learning Operations)** - Культурная и техническая практика, объединяющая разработку моделей машинного обучения (ML) и их эксплуатацию для надежного и эффективного развертывания в production.

- **Mob Programming** - практика, когда вся команда разработчиков работает над одним кодом за одним компьютером.

- **Mocking (Мокирование)** - техника тестирования, при которой реальные зависимости (например, база данных) заменяются контролируемыми имитациями.

- **Model Drift (Дрейф модели)** - Ухудшение качества прогнозов модели со временем из-за изменений в реальных данных, на которых модель больше не актуальна.

- **Model Registry (Реестр моделей)** - Централизованное хранилище для управления версиями обученных моделей, их метаданными и жизненным циклом (стадиями от разработки до продакшена).

- **Model Serving (Обслуживание моделей)** - Процесс развертывания и предоставления доступа к ML-модели для получения прогнозов (инференса) через API или другие интерфейсы.

- **Monolith (Монолит)** - традиционная архитектура приложения, где все компоненты тесно связаны и развертываются как единое целое.

- **MongoDB (MongoDB, Inc.)** - документоориентированная NoSQL база данных.

- **Multi-Factor Authentication (MFA)** - метод аутентификации, требующий предоставления двух или более независимых доказательств (факторов) для подтверждения личности.

- **Multithreading (Многопоточность)** - способность центрального процессора выполнять несколько потоков команд одновременно.

- **MySQL (Oracle)** - одна из самых популярных в мире систем управления реляционными базами данных с открытым исходным кодом.

### N

- **Nameserver (NS)** - сервер в системе DNS, отвечающий за преобразование доменных имен в IP-адреса.

- **Namespace (Пространство имен)** - изолированная область (в Kubernetes, Linux, программировании) для логической группировки ресурсов и предотвращения конфликтов имен.

- **NAT (Network Address Translation)** - метод преобразования IP-адресов в заголовках пакетов при их передаче через маршрутизатор или межсетевой экран.

- **Natural Language Processing (NLP)** - область искусственного интеллекта, позволяющая машинам понимать и обрабатывать человеческий язык.

- **Neptune.ai** - Хранилище для MLOps, предназначенное для регистрации, хранения, организации и сравнения всех метаданных, генерируемых в ходе жизненного цикла модели.

- **Nessus (Tenable)** - проприетарный инструмент для сканирования уязвимостей сетей и систем.

- **Net Promoter Score (NPS)** - метрика лояльности клиентов, измеряющая их готовность рекомендовать продукт или компанию.

- **Netcat** - утилита командной строки для чтения и записи данных через сетевые соединения по протоколам TCP/IP.

- **NetFlow** - протокол от Cisco для сбора и анализа статистики IP-трафика в сетях.

- **Network Interface Card (NIC)** - аппаратный компонент (сетевая карта), обеспечивающий подключение компьютера к сети.

- **Neural Network (Нейронная сеть)** - вычислительная модель, вдохновленная биологическими нейронными сетями мозга, основа алгоритмов глубокого обучения.

- **Nexus Repository Manager (Sonatype)** - Репозиторий для хранения и управления артефактами сборки (Java-пакеты, Docker-образы, npm-пакеты и другие).

- **Nginx (F5 Networks)** - высокопроизводительный веб-сервер, часто используемый как обратный прокси, балансировщик нагрузки и кэширующий сервер.

- **Nix/NixOS (NixOS Foundation)** - система управления пакетами и операционная система.

- **Nmap (Network Mapper)** - утилита с открытым исходным кодом для исследования сети и проверки безопасности.

- **Node.js (OpenJS Foundation)** - среда выполнения JavaScript на стороне сервера, построенная на движке V8 от Chrome.

- **NoSQL (Not Only SQL)** - широкий класс систем управления базами данных, не использующих реляционную модель (MongoDB, Cassandra).

- **npm (Node Package Manager)** - менеджер пакетов и реестр для языка программирования JavaScript, стандарт для Node.js.

- **NTFS (New Technology File System)** - стандартная файловая система для операционных систем семейства Windows NT.

- **NTLM (NT LAN Manager)** - проприетарный пакет протоколов аутентификации от Microsoft.

- **NTP (Network Time Protocol)** - сетевой протокол для синхронизации внутренних часов компьютеров с точными источниками времени.

- **NVM (Node Version Manager)** - утилита командной строки для управления несколькими версиями Node.js на одной машине.

- **NVIDIA CUDA (NVIDIA)** - параллельная вычислительная платформа и API, позволяющая использовать графические процессоры (GPU) для общих вычислений.

### O

- **OAuth** - открытый стандарт авторизации для безопасного делегирования доступа к ресурсам.

- **Object-Relational Mapping (ORM)** - технология, связывающая базы данных с объектно-ориентированными языками программирования.

- **Observability (Наблюдаемость)** - характеристика системы, позволяющая по внешним данным (метрики, логи, трейсы) понять её внутреннее состояние.

- **OKR (Objectives and Key Results)** - система постановки и достижения целей, синхронизирующая работу команды.

- **OLAP (Online Analytical Processing)** - технология для сложного аналитического анализа многомерных данных.

- **OLTP (Online Transaction Processing)** - технология для обработки оперативных транзакций в реальном времени.

- **OOM Killer (Out-Of-Memory Killer)** - процесс в ядре Linux, который завершает приложения для предотвращения исчерпания памяти.

- **OpenMetadata** - Единая платформа для управления и каталогизации метаданных, объединяющая информацию о данных, их происхождении, качестве и владельцах из разных источников.

- **OpenShift (Red Hat)** - платформа для контейнерных приложений на основе Kubernetes.

- **Open Source (Опенсорс)** - программное обеспечение, исходный код которого открыт для использования, изучения и изменения.

- **OpenTelemetry (Cloud Native Computing Foundation)** - стандарт для генерации, сбора и экспорта телеметрии (метрики, логи, трассировки).

- **OpenTofu (Linux Foundation)** - инструмент Infrastructure as Code, развивающийся как независимый форк Terraform.

- **OpenVPN** - система с открытым исходным кодом для создания зашифрованных VPN-соединений.

- **Operating System (OS)** - базовое программное обеспечение, управляющее ресурсами компьютера и обеспечивающее работу приложений.

- **Oracle Database (Oracle)** - коммерческая реляционная система управления базами данных корпоративного класса.

- **OSI Model (Open Systems Interconnection)** - эталонная модель для описания сетевых протоколов и коммуникаций в виде семи уровней.

- **Overfitting (Переобучение)** - проблема в машинном обучении, когда модель слишком точно подстраивается под обучающие данные и плохо работает на новых.

### P

- **PaaS (Platform as a Service)** - модель облачных вычислений, предоставляющая платформу для разработки, запуска и управления приложениями.

- **Pachyderm (Pachyderm)** - платформа версионирования данных и конвейеров для воспроизводимого ML.

- **Package Manager (Менеджер пакетов)** - инструмент для автоматизации установки, обновления и удаления программных пакетов (apt, yum, npm).

- **Packet (Пакет)** - единица данных, передаваемая по сети.

- **Pandas** - популярная библиотека Python для анализа и обработки данных.

- **Parser (Парсер)** - программа или часть кода, анализирующая входные данные (например, текст, XML, JSON) для извлечения структурированной информации.

- **PCI Express (PCIe)** - высокоскоростной стандарт подключения для внутренних компонентов компьютера (видеокарт, SSD).

- **Penetration Testing (Pentest)** - метод оценки безопасности системы путем моделирования атак злоумышленника.

- **Perl** - высокоуровневый интерпретируемый язык общего назначения, известный мощной обработкой текста.

- **PHP** - популярный серверный язык сценариев для веб-разработки.

- **Pipeline (Пайплайн)** - последовательность автоматизированных шагов для обработки данных, сборки или развертывания приложений.

- **PostgreSQL** - мощная объектно-реляционная система управления базами данных с открытым исходным кодом.

- **Prefect (Prefect Technologies)** - платформа для оркестрации рабочих процессов данных и ML.

- **Product Owner (PO)** - роль в Scrum, отвечающая за максимизацию ценности продукта и управление бэклогом.

- **Production (Прод, Продакшн)** - рабочая, production-среда, где приложение доступно конечным пользователям.

- **Project Manager (PM, Проджект-менеджер)** - специалист, отвечающий за планирование, исполнение и завершение проекта.

- **PVC (PersistentVolumeClaim)** - В Kubernetes: запрос пользователя (пода) на хранилище определенного размера и характеристик, абстрагирующий его от конкретного физического диска.

- **Python** - интерпретируемый высокоуровневый язык программирования общего назначения, известный простым синтаксисом и широким применением.

### Q

- **QA (Quality Assurance)** - обеспечение качества, процесс, направленный на предотвращение дефектов в разрабатываемом продукте.

- **QEMU (Quick Emulator)** - свободный эмулятор с открытым исходным кодом для аппаратного обеспечения.

- **QoS (Quality of Service)** - технологии управления трафиком для обеспечения заданного уровня производительности сети.

- **QR Code (Quick Response Code)** - тип матричного штрихкода, предназначенный для быстрого считывания и хранения информации.

- **Quadratic Time Complexity O(n²)** - класс сложности алгоритма, где время выполнения пропорционально квадрату размера входных данных.

- **Query (Запрос)** - запрос на получение или изменение данных, например, в базе данных (SQL-запрос) или к API.

- **Query Optimization** - процесс выбора наиболее эффективного способа выполнения запроса к базе данных.

- **Queue (Очередь)** - структура данных, работающая по принципу FIFO (First-In-First-Out), часто используется для управления задачами.

- **Qwik** - современный фреймворк для веб-приложений, ориентированный на максимальную производительность за счёт ленивой загрузки.

### R

- **RabbitMQ** - Система обмена сообщениями (message broker), реализующая протокол AMQP для связи между микросервисами и приложениями.

- **RAG (Retrieval Augmented Generation)** - Подход в генеративном ИИ, при котором ответы модели дополняются информацией из внешних источников (например, базы документов).

- **Rancher (SUSE)** - платформа для управления несколькими кластерами Kubernetes.

- **Random Forest (Случайный лес)** - Алгоритм машинного обучения, использующий ансамбль (множество) решающих деревьев для повышения точности и устойчивости прогнозов.

- **Ransomware** - Вредоносное программное обеспечение, которое шифрует данные жертвы и требует выкуп за их расшифровку.

- **RDS (Relational Database Service)** - Управляемый облачный сервис для работы с реляционными базами данных, например, от AWS или других провайдеров.

- **React** - Популярная JavaScript-библиотека с открытым исходным кодом для создания пользовательских интерфейсов, особенно одностраничных приложений.

- **Real-Time Processing (Обработка в реальном времени)** - Непрерывная обработка потока данных сразу после их поступления для получения немедленных результатов.

- **Recall** - Метрика в машинном обучении, показывающая, какую долю реальных положительных случаев модель смогла правильно идентифицировать.

- **Reconciliation (Сверка)** - В GitOps --- процесс, при котором система автоматически приводит состояние инфраструктуры или приложения в соответствие с декларативным описанием в репозитории.

- **Redis** - Высокопроизводительная база данных типа "ключ-значение" в памяти, часто используемая как кэш, брокер сообщений или хранилище сессий.

- **Regression Testing (Регрессионное тестирование)** - Тип тестирования, проверяющий, что новые изменения в коде не нарушили существующую функциональность.

- **Regularization (Регуляризация)** - Техника в машинном обучении для предотвращения переобучения модели путем добавления штрафа за её излишнюю сложность.

- **Reinforcement Learning (Обучение с подкреплением)** - Тип машинного обучения, при котором алгоритм (агент) учится принимать решения через взаимодействие со средой и получение наград/штрафов.

- **Release (Релиз)** - Версия программного обеспечения, готовая для развертывания в production-среде.

- **Replica Set (Набор реплик)** - В Kubernetes --- объект, обеспечивающий поддержание указанного количества идентичных подов (Pods) в рабочем состоянии.

- **Replication (Репликация)** - Технология создания и поддержания копий (реплик) данных на нескольких серверах для повышения доступности, отказоустойчивости и распределения нагрузки на чтение.

- **Repository (Репозиторий)** - Хранилище для кода (Git), пакетов программного обеспечения (Docker Registry) или артефактов сборки (Artifactory).

- **Reproducibility (Воспроизводимость)** - Ключевое требование MLOps: способность гарантированно повторить весь процесс обучения модели (код, данные, параметры, окружение) и получить идентичный результат.

- **REST API** - Архитектурный стиль для проектирования сетевых API, использующий стандартные HTTP-методы (GET, POST, PUT, DELETE).

- **Retry (Повторная попытка)** - Паттерн устойчивости в распределенных системах, при котором неудачный запрос автоматически повторяется через некоторое время.

- **Reverse Proxy (Обратный прокси)** - Сервер, который принимает запросы от клиентов и перенаправляет их на внутренние серверы, часто используемый для балансировки нагрузки, кэширования и обеспечения безопасности.

- **RLHF (Reinforcement Learning from Human Feedback)** - Метод дообучения больших языковых моделей (LLM), при котором модель оптимизируется на основе оценок, данных человеком.

- **Rollback (Откат)** - Процесс возврата системы или приложения к предыдущей, стабильной версии после неудачного обновления или обнаружения критической ошибки.

- **Rolling Update (Постепенное обновление)** - Стратегия развертывания в Kubernetes, при которой поды с новой версией приложения постепенно заменяют поды со старой версией, минимизируя простой.

- **Root Cause Analysis (Анализ первопричины)** - Процесс поиска и устранения основной причины проблемы, а не только её симптомов. Ключевая практика в SRE.

- **RPA (Robotic Process Automation)** - Технология автоматизации повторяющихся, рутинных бизнес-процессов с помощью программных "роботов".

- **RSA** - Широко используемый алгоритм асимметричного шифрования, применяемый для безопасной передачи данных и цифровых подписей.

- **RStudio** - Интегрированная среда разработки (IDE) для языка программирования R, популярная среди статистиков и специалистов по анализу данных.

- **Ruby on Rails** - Веб-фреймворк, написанный на языке Ruby, следующий принципам "соглашение над конфигурацией" (Convention over Configuration).

- **Runtime (Среда выполнения)** - Программное окружение, в котором исполняется код (например, JRE для Java, .NET CLR, среда выполнения Python).

### S

- **SaaS (Software as a Service)** - Модель лицензирования, при которой программное обеспечение предоставляется по подписке и доступно через интернет.

- **SAML (Security Assertion Markup Language)** - Стандарт на основе XML для обмена данными аутентификации и авторизации между доменами, часто используется для реализации единого входа (SSO).

- **SAS (Statistical Analysis System)** - Проприетарное программное обеспечение для продвинутой аналитики, управления бизнес-данными и прогнозирования.

- **Sass (Syntactically Awesome Stylesheets)** - Препроцессор CSS, расширяющий возможности стандартного CSS переменными, вложенностью, миксинами и другими функциями.

- **SAST (Static Application Security Testing)** - Статический анализ безопасности приложений, проверка исходного кода на наличие уязвимостей.

- **S3 (Amazon Simple Storage Service)** - Объектное хранилище данных от AWS, предлагающее масштабируемость, доступность и безопасность.

- **Scala** - Язык программирования, сочетающий объектно-ориентированные и функциональные парадигмы, работает на JVM. Популярен в экосистеме Apache Spark.

- **Schema (Схема)** - Структурное описание данных, определяющее их организацию, типы и отношения.

- **Scikit-learn** - Популярная библиотека машинного обучения для Python, предоставляющая простые и эффективные инструменты для анализа данных и моделирования.

- **Scrum** - Agile-фреймворк для управления проектами, основанный на итеративных циклах разработки (спринтах) и ролях (Владелец продукта, Scrum-мастер, Команда разработки).

- **SDLC (Software Development Life Cycle)** - Жизненный цикл разработки программного обеспечения --- процесс от планирования до вывода из эксплуатации.

- **SDN (Software-Defined Networking)** - Подход к управлению сетями, при котором плоскость управления отделяется от плоскости данных и программируется централизованно.

- **Secrets Management (Управление секретами)** - Практика безопасного хранения и управления конфиденциальной информацией (пароли, ключи API, токены), например, с помощью HashiCorp Vault.

- **Seldon Core (Seldon)** - Платформа с открытым исходным кодом для развертывания, масштабирования, мониторинга и управления моделями машинного обучения в Kubernetes.

- **Self-Healing (Самовосстановление)** - Способность системы автоматически обнаруживать сбои и предпринимать действия по их устранению без вмешательства человека (например, перезапуск упавшего контейнера в Kubernetes).

- **SELinux (Security-Enhanced Linux)** - Модуль безопасности ядра Linux, предоставляющий механизм принудительного контроля доступа (MAC).

- **Semantic Versioning (Семантическое версионирование)** - Соглашение о нумерации версий ПО в формате `MAJOR.MINOR.PATCH`, где каждое число увеличивается в зависимости от характера изменений.

- **Sentinel (HashiCorp)** - Фреймворк для встраивания политик управления и безопасности в конвейеры HashiCorp.

- **Serverless (Бессерверная архитектура)** - Модель облачных вычислений, в которой провайдер динамически управляет выделением ресурсов, а пользователь оплачивает фактическое время выполнения кода (функций).

- **Service (Сервис)** - 1) В Kubernetes --- абстракция для доступа к набору подов. 2) В микросервисной архитектуре --- независимый компонент, реализующий конкретную бизнес-функцию.

- **Service Level Agreement (SLA)** - Формальный договор между поставщиком и потребителем услуги, определяющий гарантированные уровни её доступности и производительности.

- **Service Level Objective (SLO)** - Внутренние целевые показатели надежности сервиса, которые должны быть строже SLA для их гарантированного выполнения.

- **Service Mesh (Сервисная сеть)** - Специализированный инфраструктурный слой (например, Istio, Linkerd) для управления взаимодействием микросервисов (безопасность, наблюдение, маршрутизация).

- **Session (Сессия)** - Временное состояние взаимодействия между пользователем (клиентом) и приложением (сервером), сохраняющее контекст между запросами.

- **SFTP (SSH File Transfer Protocol)** - Сетевой протокол для безопасной передачи файлов поверх защищенного канала SSH.

- **SHA (Secure Hash Algorithm)** - Семейство криптографических хеш-функций, используемых для создания цифровых отпечатков данных (например, SHA-256).

- **Shadow Mode (Теневой режим)** - Режим запуска новой модели параллельно со старой, где новая модель делает прогнозы "в тени", но не влияет на решения, что позволяет оценить её поведение на реальных данных.

- **Sharding (Шардирование)** - Метод горизонтального разделения данных в базе данных: строки одной таблицы распределяются по разным серверам (шардам) на основе ключа шардирования.

- **SIEM (Security Information and Event Management)** - Система для сбора, анализа и корреляции данных о безопасности из различных источников в реальном времени.

- **Singularity (Sylabs)** - Контейнерная платформа, разработанная для высокопроизводительных вычислений (HPC) и научных рабочих нагрузок.

- **SLA / SLO / SLI** - **SLA** --- договор с клиентом, **SLO** --- внутренняя цель, **SLI** --- конкретная метрика для измерения надежности (например, процент успешных запросов).

- **Slack (Salesforce)** - платформа для командной коммуникации.

- **SMTP (Simple Mail Transfer Protocol)** - Стандартный сетевой протокол для отправки электронной почты.

- **Snake Case (Змеиный_регистр)** - Стиль написания составных слов, при котором слова разделяются символом подчеркивания, а все буквы обычно строчные (например, `my_variable_name`).

- **Snowflake (Snowflake Inc.)** - облачное хранилище данных с разделением вычислительных ресурсов и хранения.

- **Snyk** - Платформа для безопасной разработки, помогающая находить и устранять уязвимости в зависимостях, контейнерах и инфраструктуре как код.

- **SOAP (Simple Object Access Protocol)** - Протокол обмена структурированными сообщениями в распределенных вычислениях, использующий XML.

- **Software Supply Chain (Цепочка поставок ПО)** - Совокупность процессов, инструментов и компонентов, участвующих в создании и доставке программного обеспечения.

- **SonarQube** - Платформа для непрерывного анализа качества кода, выявления ошибок, уязвимостей и "запахов кода".

- **Spark (Apache PySpark)** - Унифицированный аналитический распределённый движок для обработки больших данных, поддерживающий SQL, потоковую обработку, машинное обучение и графовые вычисления.

- **Splunk** - Платформа для мониторинга и анализа данных машинного происхождения (логи, метрики), широко используемая в IT-операциях и безопасности.

- **SQL (Structured Query Language)** - Декларативный язык программирования, предназначенный для управления данными в реляционных базах данных и работы с ними.

- **SQL Injection** - Распространенный метод кибератаки, при котором злоумышленник внедряет вредоносный SQL-код во входные данные приложения.

- **SSD (Solid State Drive)** - Накопитель на основе флеш-памяти, не имеющий движущихся частей, что обеспечивает высокую скорость чтения/записи и ударопрочность.

- **SSH (Secure Shell)** - Криптографический сетевой протокол для безопасного удаленного управления операционными системами и туннелирования TCP-соединений.

- **SSL/TLS** - Криптографические протоколы, обеспечивающие защищённую передачу данных по сети (TLS --- более новая и безопасная версия SSL).

- **Stack (Стек)** - 1) В программировании --- структура данных, работающая по принципу LIFO. 2) Набор взаимосвязанных технологий (например, LAMP-стек).

- **Staging (Стейдинг-среда)** - Предпродакшенная среда, максимально приближенная к production, используется для финального тестирования и проверки сборок.

- **Stateful / Stateless** - **Stateful** --- приложение, хранящее состояние сессии на сервере. **Stateless** --- не хранящее (каждый запрос независим), что упрощает масштабирование.

- **StorageClass** - В Kubernetes: "класс" или тип хранилища (например, SSD, медленный HDD), который администратор кластера определяет для динамического выделения PVC.

- **Streaming (Потоковая обработка)** - Непрерывная обработка бесконечных потоков данных в реальном времени (например, с помощью Apache Kafka или Apache Flink).

- **Stress Testing (Стресс-тестирование)** - Тип тестирования, проверяющий устойчивость системы под экстремальными нагрузками, превышающими нормальные условия эксплуатации.

- **Subnet (Подсеть)** - Логическое разделение IP-сети на несколько более мелких сетевых сегментов.

- **Supervisor** - Системный демон в Unix-подобных ОС, предназначенный для управления процессами, их автоматического перезапуска при сбоях.

- **Supply Chain Attack (Атака на цепочку поставок)** - Киберинцидент, при котором злоумышленник компрометирует систему через уязвимость в сторонней библиотеке или инструменте разработки.

- **SVM (Support Vector Machine)** - Алгоритм машинного обучения для классификации и регрессии, эффективно работающий в высокоразмерных пространствах признаков.

- **Swagger / OpenAPI** - Спецификация и набор инструментов для описания, создания и документирования RESTful API.

- **Syslog** - Стандартный протокол для отправки и сбора логов в Unix-подобных системах и сетевых устройствах.

### T

- **Tableau (Salesforce)** - Платформа для бизнес-аналитики и интерактивной визуализации данных, которая позволяет создавать отчеты и дашборды.

- **Terraform (HashiCorp)** - Ведущий инструмент для управления инфраструктурой как код (IaC), позволяет безопасно и эффективно создавать и изменять облачные ресурсы с помощью декларативных конфигурационных файлов.

- **Tekton (Cloud Native Computing Foundation)** - платформа для создания CI/CD-систем в Kubernetes.

- **TensorFlow (Google)** - Открытая платформа машинного обучения (ML) для создания и обучения нейронных сетей, особенно в областях глубокого обучения и компьютерного зрения.

- **Test-Driven Development (TDD)** - Методология разработки программного обеспечения, при которой тесты пишутся до кода, что способствует созданию более надежного и простого в поддержке кода.

- **Time to Restore (MTTR)** - Ключевая метрика в DevOps и SRE, означающая среднее время, необходимое для восстановления работоспособности сервиса после сбоя. Отражает надежность и оперативность реагирования команды.

- **TLS/SSL** - Криптографические протоколы, обеспечивающие безопасность связи по компьютерным сетям. TLS - более современный и безопасный преемник SSL. Используются для шифрования HTTP-трафика (HTTPS), почты и многого другого.

- **Traefik (Traefik Labs)** - Обратный прокси и балансировщик нагрузки для микросервисов, автоматически обнаруживающий новые сервисы и их конфигурацию.

- **Trino (ранее PrestoSQL)** - Высокопроизводительный распределенный SQL-движок для выполнения интерактивных аналитических запросов к данным из разнородных источников.

- **Trunk-Based Development** - Стратегия управления исходным кодом, при которой разработчики часто делают небольшие коммиты напрямую в основную ветку (trunk), что является основой для быстрой непрерывной интеграции (CI).

- **Two-Factor Authentication (2FA)** - Метод аутентификации, при котором для подтверждения личности пользователя требуется два разных типа доказательств (например, пароль и код из SMS).

### U

- **UAT (User Acceptance Testing)** - Финальная фаза тестирования, в ходе которой конечные пользователи проверяют, соответствует ли система бизнес-требованиям и готова ли к работе в реальных условиях.

- **Ubuntu (Canonical)** - Популярный дистрибутив операционной системы Linux на базе Debian, известный удобством использования и широкой поддержкой.

- **UML (Unified Modeling Language)** - Стандартизированный графический язык моделирования для визуализации, спецификации, конструирования и документирования артефактов программных систем.

- **Unit Testing (Модульное тестирование)** - Уровень тестирования программного обеспечения, при котором проверяются отдельные, изолированные части приложения (юниты), обычно функции или методы.

- **URI (Uniform Resource Identifier)** - Строка символов, используемая для идентификации какого-либо ресурса (например, документа или сервиса) в интернете. URL - это частный случай URI, который также указывает местоположение и способ получения ресурса.

- **UUID (Universally Unique Identifier)** - Стандарт идентификации, используемый в создании программного обеспечения. UUID - это 128-битное число, предназначенное для практической гарантии уникальности во времени и пространстве.

### V

- **Vagrant (HashiCorp)** - Инструмент для создания и управления средами виртуальных машин в едином рабочем процессе. Позволяет легко настраивать воспроизводимые окружения для разработки и тестирования.

- **Vault (HashiCorp)** - Инструмент для безопасного хранения, доступа и управления секретами, такими как токены, пароли, сертификаты и ключи шифрования.

- **Velero** - Инструмент для резервного копирования, миграции и аварийного восстановления ресурсов и данных в Kubernetes.

- **Version Control System (VCS)** - Программное обеспечение для управления изменениями в исходном коде и других файлах с течением времени (например, Git, Subversion).

- **Vertical Scaling (Вертикальное масштабирование)** - Стратегия увеличения производительности системы за счет добавления ресурсов (больше CPU, RAM, дискового пространства) на существующий сервер. Противопоставляется горизонтальному масштабированию (добавлению большего количества серверов).

- **Virtual Machine (VM)** - Программная эмуляция физического компьютера, которая запускает свою собственную операционную систему и приложения на хост-машине с использованием гипервизора.

- **Visual Studio Code (Microsoft)** - редактор исходного кода с поддержкой расширений.

- **Vue.js** - Прогрессивный JavaScript-фреймворк с открытым исходным кодом для создания пользовательских интерфейсов и одностраничных приложений (SPA), известный своей простотой интеграции и гибкостью.

### W

- **WAF (Web Application Firewall)** - Брандмауэр веб-приложений, фильтрующий и блокирующий вредоносный HTTP-трафик для защиты веб-приложений от атак.

- **WAN (Wide Area Network)** - Глобальная вычислительная сеть, охватывающая большие географические области (например, сеть между офисами компании).

- **Wasm (WebAssembly)** - Переносимый бинарный формат кода, позволяющий выполнять высокопроизводительные программы на стороне клиента (в браузере) или сервера.

- **Waterfall Model (Водопадная модель)** - Последовательная методология разработки ПО, где каждая фаза (сбор требований, проектирование, разработка, тестирование, внедрение, поддержка) следует строго за предыдущей.

- **Webhook (Вебхук)** - Механизм для отправки автоматических уведомлений (HTTP-запросов) от одного приложения к другому при наступлении определенного события.

- **Webpack** - Статический сборщик модулей для JavaScript-приложений, превращающий множество файлов (JS, CSS, изображения) в один или несколько оптимизированных бандлов.

- **WebRTC (Web Real-Time Communication)** - Технология для передачи потокового аудио и видео и обмена данными между браузерами в режиме реального времени без плагинов.

- **WebSocket** - Протокол связи, обеспечивающий постоянное, двунаправленное соединение между клиентом и сервером поверх одного TCP-соединения.

- **Weights & Biases (W&B)** - Платформа для отслеживания экспериментов, версионирования данных и моделей, визуализации и совместной работы команд ML.

- **Wi-Fi (Wireless Fidelity)** - Технология беспроводной локальной сети (WLAN), основанная на стандартах IEEE 802.11.

- **Wildcard Certificate (Wildcard-сертификат)** - SSL/TLS сертификат, который обеспечивает безопасность одного доменного имени и всех его поддоменов.

- **Windows Server (Microsoft)** - Линейка серверных операционных систем от Microsoft для управления сетями, данными и приложениями.

- **Wireframing (Вайрфрейминг)** - Процесс создания схематичного макета (скелета) пользовательского интерфейса для визуализации структуры и компоновки элементов.

- **WordPress** - Популярная система управления контентом (CMS) с открытым исходным кодом, написанная на PHP, используется для создания блогов, сайтов и интернет-магазинов.

- **Worm (Компьютерный червь)** - Тип вредоносного программного обеспечения, которое самостоятельно распространяется по сети, используя уязвимости, и наносит ущерб.

- **WSDL (Web Services Description Language)** - Язык описания веб-служб на основе XML, используемый для описания функциональности, предоставляемой SOAP-веб-сервисом.

- **WYSIWYG (What You See Is What You Get)** - Принцип редактирования, при котором содержание (например, текст или страница) во время редактирования выглядит максимально похоже на конечный результат.

### X

- **X.509** - Стандарт, определяющий формат цифровых сертификатов с открытым ключом, которые используются в инфраструктуре открытых ключей (PKI).

- **XAI (Explainable AI, Объяснимый ИИ)** - Набор методов и технологий в искусственном интеллекте, позволяющих понять, интерпретировать и объяснить решения и прогнозы, выдаваемые моделями машинного обучения.

- **XAMPP** - Свободный кроссплатформенный веб-серверный пакет, включающий Apache, MariaDB, PHP и Perl, предназначенный для локальной разработки и тестирования веб-приложений.

- **Xen** - Монолитный гипервизор с открытым исходным кодом, позволяющий запускать несколько изолированных операционных систем (гостевых ОС) на одном физическом оборудовании (хосте).

- **XGBoost (Extreme Gradient Boosting)** - Высокопроизводительная, масштабируемая библиотека для машинного обучения с градиентным бустингом, широко используемая в соревнованиях по data science.

- **XHTML (Extensible HyperText Markup Language)** - Язык разметки, представляющий собой переформулировку HTML 4 в качестве приложения XML; более строгий и чистый, чем HTML.

- **XML (Extensible Markup Language)** - Расширяемый язык разметки, использующий теги для структурирования, хранения и передачи данных в виде текста, понятного человеку и машине.

- **XML Schema (XSD)** - Язык описания структуры и ограничений XML-документов, более мощная альтернатива DTD.

- **XMPP (Extensible Messaging and Presence Protocol)** - Открытый протокол для обмена сообщениями и информацией о присутствии в реальном времени, основанный на XML (ранее известный как Jabber).

- **XOR (Exclusive OR, Исключающее ИЛИ)** - Логическая операция, результат которой истинен только тогда, когда ровно один из операндов истинен.

- **XPath (XML Path Language)** - Язык запросов для выбора узлов из XML-документа, также используемый для вычисления значений (строк, чисел, логических значений) из содержимого XML-документа.

- **XQuery** - Язык запросов, предназначенный для извлечения и манипулирования данными, хранящимися в XML-формате.

- **XSS (Cross-Site Scripting, Межсайтовый скриптинг)** - Распространенный тип уязвимости веб-приложений, при котором злоумышленник внедряет вредоносные скрипты на веб-страницы, просматриваемые другими пользователями.

- **XSL (Extensible Stylesheet Language)** - Семейство языков преобразования и форматирования XML-документов, включающее XSLT, XPath и XSL-FO.

- **XSLT (XSL Transformations)** - Язык преобразования XML-документов в другие XML-документы или в другие форматы, такие как HTML или обычный текст.

### Y

- **YAML (YAML Ain't Markup Language)** - Человеко-читаемый язык сериализации данных, часто используемый для написания конфигурационных файлов (например, в Ansible, Kubernetes, Docker Compose).

- **YANG (Yet Another Next Generation)** - Язык моделирования данных, используемый для описания конфигурации и состояния сетевых устройств в программно-конфигурируемых сетях (SDN).

- **Yarn (Yet Another Resource Negotiator) (Apache)** - Планировщик ресурсов и менеджер рабочих нагрузок для кластеров Hadoop, отвечающий за распределение вычислительных ресурсов между приложениями.

- **Yarn (Package Manager)** - Менеджер пакетов для JavaScript, разработанный компанией Facebook как альтернатива npm, с акцентом на скорость и надежность.

- **Y2K Bug (Проблема 2000 года)** - Компьютерная ошибка, связанная с обработкой дат в XX и XXI веках, когда из-за экономии памяти год записывался двумя цифрами (например, "99" вместо "1999").

- **YOLO (You Only Look Once)** - Алгоритм и семейство архитектур нейронных сетей для обнаружения объектов на изображениях и видео в реальном времени, отличающееся высокой скоростью работы.

- **YSlow** - Инструмент для анализа производительности веб-страниц, который оценивает страницы на основе определённого набора правил для высокопроизводительных веб-сайтов.

### Z

- **Zabbix** - Корпоративное программное обеспечение с открытым исходным кодом для мониторинга сетей, серверов, приложений и сервисов, собирающее различные метрики.

- **Zen of Python** - Коллекция из 19 принципов (афоризмов), влияющих на дизайн языка программирования Python, написанная Тима Петерсом. Доступна по команде `import this`.

- **Zero-Day (Zero-Day Vulnerability / Exploit)** - Уязвимость в программном обеспечении, неизвестная разработчику, для которой на момент обнаружения не существует исправления (патча). Также называется атакой нулевого дня.

- **Zero Trust (Zero Trust Architecture, ZTA)** - Концепция и стратегия кибербезопасности, которая предполагает, что угроза может исходить как извне, так и изнутри сети, и поэтому требует строгой проверки подлинности для каждого устройства и пользователя, пытающегося получить доступ к ресурсам.

- **ZFS (Zettabyte File System)** - Продвинутая файловая система и менеджер томов, разработанная Sun Microsystems, с поддержкой больших объёмов данных, снапшотов, копирования при записи и встроенных механизмов проверки целостности.

- **Zigbee** - Спецификация для набора протоколов высокоуровневой связи, использующих небольшие маломощные радиопередатчики для создания персональных сетей (например, для домашней автоматизации).

- **ZITADEL (ZITADEL)** - платформа управления идентификацией и доступом с открытым исходным кодом.

- **Zookeeper (Apache ZooKeeper)** - Централизованная служба для распределенных систем, предоставляющая такие функции, как обслуживание конфигурационной информации, именования, синхронизации и групповых сервисов. Часто используется с Kafka и Hadoop.

- **Zsh (Z Shell)** - Мощная командная оболочка для Unix-подобных операционных систем, которая может использоваться как интерактивная оболочка входа и как интерпретатор команд. Является основой для фреймворка Oh My Zsh.
